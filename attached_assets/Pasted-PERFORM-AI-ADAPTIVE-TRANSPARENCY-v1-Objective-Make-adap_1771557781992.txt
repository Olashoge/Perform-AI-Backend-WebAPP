PERFORM AI — ADAPTIVE TRANSPARENCY v1

Objective:
Make adaptive intelligence visible and trusted without adding new tabs or complex dashboards.

We are NOT adding new features.
We are exposing intelligence that already exists (Adaptive Engine v1 + performance summaries).

============================================================
PART 1 — “Why This Plan Looks Like This”
============================================================

On every generated plan page (Wellness, Meal Plan, Workout Plan, Daily Meal, Daily Workout):

1. If adaptiveSnapshot exists on the plan:
   - Display a subtle section titled:
       "Why this plan looks like this"
   - This section must appear below the plan title and above the content.
   - It must be collapsible (default: expanded).

2. Render 2–4 human-readable bullets derived from adaptiveSnapshot.decisions.
   Use deterministic mapping from decision codes to plain English.

   Example mapping:
   LOW_WORKOUT_ADHERENCE_SIMPLIFY →
       "Your recent workout consistency was lower, so we simplified this week."
   FATIGUE_RISK_DELOAD →
       "Energy levels were low, so this week emphasizes recovery."
   STALL_WEIGHT_LOSS_ADJUST →
       "Weight progress slowed, so calories and conditioning were adjusted."
   HIGH_WORKOUT_ADHERENCE_PROGRESS →
       "Strong consistency allowed a small progression this week."

3. Do NOT show technical variables.
   Do NOT show percentages.
   Do NOT show multipliers.
   This is trust-building, not a debug panel.

4. If no adaptiveSnapshot exists (first week ever):
   Show:
       "This plan is based on your profile and goals."

PART 2 — Weekly Trend Visibility (Inside Check-In Tab)
============================================================

Enhance the Check-In page:

1. Below the current week score card, add a compact section:

   Title: "Recent Progress"

   - Momentum state badge (Building / Maintaining / Fatigue Risk / Slipping)

       ↑ if score > previous week
       ↓ if score < previous week
       → if stable (±5 points)

3. Keep it minimal:
   - No charts
   - No heavy graphs
   - Clean, premium, aligned with existing design tokens
