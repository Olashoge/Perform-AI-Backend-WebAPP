STEP 3 — Create Constraint Engine module
Create a folder like:
- server/constraints/

Files to create:
1) server/constraints/types.ts
Define:
- UserContext: profile, preferences, schedule context, requested plan type, date range, etc.
- SafeSpec: normalized units + allowed foods/exercises + macro bounds + schedule rules
- Violation: category, severity, message, metadata
- ConstraintResult: { blocked: boolean, violations: Violation[], safeSpec: SafeSpec }

2) server/constraints/rules/age.ts
Rules:
- age < 13: BLOCK any Strength/Cardio "adult training plan" generation. Return message: "This plan is not appropriate for this age. Create a kids movement plan instead." (for v1, block; do not generate).
- age 13-15: ADJUST to remove barbell/heavy intensity patterns.
- age 16-17: WARN/ADJUST: restrict intensity, avoid advanced lifts unless experience advanced.
- age >= 60: ADJUST: joint-friendly bias, longer warmup, reduce impact.

3) server/constraints/rules/injury.ts
Convert injuries/mobility limitations into banned movement tags.
Create a simple mapping:
- ACL / knee pain -> ban: jumping/plyo, deep knee flexion loaded, high impact
- Shoulder limitation -> ban: overhead press, snatch/jerk patterns
- Lower back pain -> ban: heavy hinge, deadlift patterns; prefer supported
Return ADJUST violations with metadata: bannedTags, swapHints.

4) server/constraints/rules/equipment.ts
If equipment = home -> restrict to home-safe library
If gym -> allow full
Return ADJUST with metadata.

5) server/constraints/rules/nutrition.ts
Clamp calorie targets:
- weight loss: cap deficit max 25% or max -700 kcal/day
- muscle gain: cap surplus +250 to +500 kcal/day
- age < 18: do not allow aggressive deficits/surplus; WARN/ADJUST
Hard-block allergens and avoid foods.

6) server/constraints/rules/schedule.ts
Prevent overlapping active Wellness Plans for same date range unless the user explicitly replaced it.
Ensure meal plan dates align with workout plan dates.

7) server/constraints/engine.ts (or index.ts)
Combine all rules:
- Start from an initial SafeSpec based on profile + request
- Apply each rule to mutate SafeSpec and add violations
- Determine blocked if any BLOCK
Return ConstraintResult
