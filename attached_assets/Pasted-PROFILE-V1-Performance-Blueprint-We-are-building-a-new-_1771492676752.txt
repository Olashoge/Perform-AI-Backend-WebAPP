PROFILE V1 (Performance Blueprint)

We are building a new foundational feature called Profile (Performance Blueprint).

This must become the single source of truth for user context and must be required before generating any new Wellness Plan.

Do NOT refactor unrelated code. Implement cleanly and integrate into existing architecture.

1️⃣ DATABASE

Create a new table:

user_profile

Fields:

id (uuid, primary key)

userId (foreign key → users.id, unique, required)

age (integer, required, must be >= 18)

sex (string, nullable)

heightCm (integer, nullable)

weightKg (float, required)

targetWeightKg (float, nullable)

primaryGoal (string, required)

trainingExperience (enum: 'beginner' | 'intermediate' | 'advanced', required)

Health Context

injuries (json array, default [])

mobilityLimitations (json array, default [])

chronicConditions (json array, default [])

Lifestyle Context

sleepHours (float, nullable)

stressLevel (enum: 'low' | 'moderate' | 'high', nullable)

activityLevel (enum: 'sedentary' | 'moderate' | 'active', nullable)

availableTrainingDays (integer, nullable)

sessionDurationMinutes (integer, nullable)

Nutrition Context

allergies (json array, default [])

intolerances (json array, default [])

religiousRestrictions (json array, default [])

appetiteLevel (enum: 'low' | 'normal' | 'high', nullable)

spicePreference (enum: 'mild' | 'medium' | 'spicy', nullable)

Metadata

createdAt

updatedAt

Constraints:

One profile per user (unique userId)

Age must be >= 18 (hard validation at API level)

Run migration safely.

2️⃣ API ROUTES

Create:

GET /api/profile
POST /api/profile
PUT /api/profile

Behavior:

If profile does not exist → GET returns null

POST creates profile

PUT updates profile

Validate:

age required and >= 18

weightKg required

primaryGoal required

trainingExperience required

Return structured validation errors.
