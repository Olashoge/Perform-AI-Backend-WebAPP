3) SCHEDULING ENGINE + CONFLICT-AWARE DATE PICKERS
------------------------------------------
Rules:
- Meal plans must not overlap other meal plans (any day with at least 1 scheduled meal instance counts as occupied).
- Workout plans must not overlap other workout plans.
- Goal plans must not overlap existing goal date ranges IF the goal includes the same plan types being created.

- For each plan type, occupied days are derived from NON-deleted plans with startDate != null expanded across 7 days.

  - if user picks blocked date, auto-suggest next available valid start date and allow one-click “Use suggested date”.

Important:
- Conflicts are evaluated separately for meals and workouts.
- For Goal Wizard creating both, startDate must be valid for BOTH simultaneously (intersection of availability windows).

Also:
- Add startDate picker to “New Meal Plan” and “New Workout Plan” pages as well (not just goal wizard).
- Those pickers must block dates only for their respective plan types.

- Calendar should show 👍 or 👎 next to meal/workout items when liked/disliked.

  - Do NOT automatically add ingredients to Avoided.
  - Instead open a modal:
    “You disliked this meal. What would you like to do?”

      B) “Avoid main ingredient in future” (shows ingredient name; requires confirm)
      C) “Avoid multiple ingredients” (optional list)
      D) “Keep ingredients, change meal style” (optional note; can be stored as preference text)
- Store result accordingly:
  - Dislike stays as a meal preference.
  - Avoided only changes when user explicitly chooses it.

- Like/dislike workouts (session or workout-day item).
- Preferences page should include:
  - Meal preferences (liked/disliked/avoided)
