ADAPTIVE INTELLIGENCE RULES
Compute at end of plan (or rolling weekly if you prefer). Apply to NEXT plan’s allowances:

REWARD CONSISTENCY
- Compute combinedAdherence = avg(mealAdherence, workoutAdherence) from last plan (0–100).
- If combinedAdherence >= 80:
  -> bonus_plan_regens += 1 (cap bonus_plan_regens to +2)
- If combinedAdherence >= 90:
  -> bonus_meal_swaps_per_day += 1 AND bonus_workout_swaps_per_day += 1 (cap each bonus +2)

PREVENT “REGEN ADDICTION”
- Compute regenRate = regens_used / regens_available in last plan window.
- If regenRate > 0.80 AND combinedAdherence < 60:
  -> penalty_plan_regens -= 1 (min plan_regens_total must never go below 3)
- Cooldown rule:
  If user performs >= 3 REGEN actions in any rolling 24 hours:
    -> apply regen_cooldown_until = now + 6 hours (server-enforced)

RESPOND TO MISMATCH (HIGH DISLIKE/AVOID)
- disliked_rate_meals = disliked_meals_actions / total_meal_interactions (7d or plan window)
- disliked_rate_workouts = disliked_exercise_actions / total_workout_interactions
- If disliked_rate_meals > 25% OR avoided_ingredients_count is growing fast:
  -> bonus_meal_swaps_per_day += 1 (cap +2)
- If disliked_rate_workouts > 25% OR avoided_exercises_count grows:
  -> bonus_workout_swaps_per_day += 1 (cap +2)

STREAK PERK
- If streak_days >= 7 (check-in streak or “days completed” streak, use existing metric):
  -> grant 1 FLEX TOKEN that can be redeemed for +1 day regen (meal or workout) once within next 7 days.
  - Token expires 7 days after issuance.
