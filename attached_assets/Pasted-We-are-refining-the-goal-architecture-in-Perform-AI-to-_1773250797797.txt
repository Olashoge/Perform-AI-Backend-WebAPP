OBJECTIVE

Upgrade the goal model so it clearly separates:

PRIMARY GOAL = body outcome / main objective
SECONDARY FOCUS = training emphasis / modifier

Use this structure:

primaryGoal (required)
secondaryFocus (optional)

Allowed PRIMARY GOALS:
- Weight Loss
- Muscle Gain
- Body Recomposition
- General Fitness
- Athletic Performance

Allowed SECONDARY FOCUS values:
- Strength
- Endurance
- Mobility
- Energy & Focus
- Recovery

This must be reflected in:
- user/profile schema
- profile read/update APIs
- onboarding/profile save logic
- wellness plan creation payloads where relevant
- plan generator inputs
- any backend summary/serialization logic currently using goal

--------------------------------------------------
SECTION 1 — DATA MODEL UPDATE

Audit the current schema and identify everywhere the app stores or expects:
- goal

Replace or migrate to:

primaryGoal: string
secondaryFocus: string | null

Requirements:
1. primaryGoal is required
2. secondaryFocus is optional
3. secondaryFocus must be one of the allowed focus values
4. primaryGoal must be one of the allowed primary goal values
5. Keep backward compatibility/migration handling for existing users if needed
6. Use consistent API naming:
   - primaryGoal
   - secondaryFocus

If the database uses snake_case internally, that is fine, but API responses must use camelCase.

--------------------------------------------------
SECTION 4 — PLAN GENERATION INPUT UPDATE

Update plan generation logic so it no longer relies on a single generic goal field.

New input shape:

generatePlan(primaryGoal, secondaryFocus)

Logic structure:

basePlan = optimize(primaryGoal)

if secondaryFocus exists:
    modify(basePlan, secondaryFocus)

Interpretation model:
- primaryGoal defines the main plan objective and nutritional/training target
- secondaryFocus modifies training style, programming emphasis, and support work
- secondaryFocus is NOT a second equal goal

Examples:
- primaryGoal = Muscle Gain, secondaryFocus = Mobility
  -> muscle-building plan with mobility support
- primaryGoal = Weight Loss, secondaryFocus = Strength
  -> fat-loss plan that preserves/emphasizes strength
- primaryGoal = Body Recomposition, secondaryFocus = Endurance
  -> recomposition plan with endurance support
- primaryGoal = Athletic Performance, secondaryFocus = Recovery
  -> performance-focused plan with recovery bias

--------------------------------------------------
SECTION 5 — GOAL SEMANTICS / MAPPING

Model the categories intentionally:

PRIMARY GOAL should drive:
- calorie strategy
- macro emphasis if applicable
- training volume target
- progression bias
- recovery demand

SECONDARY FOCUS should modify:
- exercise selection
- cardio emphasis
- warmup/cooldown structure
- rest interval bias
- accessory and support work

Examples:
- Weight Loss -> deficit-oriented nutrition + sustainable resistance/cardio balance
- Muscle Gain -> surplus-oriented nutrition + hypertrophy/strength bias
- Body Recomposition -> high-protein balanced structure with body-composition focus
- General Fitness -> balanced sustainable training/nutrition
- Athletic Performance -> performance-oriented programming

Examples of secondary modifiers:
- Strength -> heavier compounds / strength emphasis
- Endurance -> more aerobic/conditioning support
- Mobility -> mobility blocks / movement quality emphasis
- Energy & Focus -> lower-friction, sustainable, mentally supportive structure
- Recovery -> fatigue management / recovery support bias
