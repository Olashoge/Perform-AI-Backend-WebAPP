ADD PREMIUM DARK MODE + THEME TOGGLE (ICON) + SETTINGS SELECTOR

GOAL
Implement first-class light/dark mode with:
1) A top-bar icon toggle (sun/moon)
2) A Settings page selector: Light / Dark / System
3) Persisted preference per user (and fallback to localStorage if user profile storage isn’t available)
4) Premium dark palette (not generic gray), aligned with our emerald + warm-neutral brand

NON-NEGOTIABLES
- Do NOT change core business logic.
- Theme switching must not affect scheduling, plans, or APIs.
- Use a single source of truth for theme state.
- Respect System preference when user selects “System”.

THEME OPTIONS
- "light"
- "dark"
- "system" (default)

PERSISTENCE
Preferred approach (in order):
A) If we have a User/Profile table or settings JSON: store themePreference there.
B) If not, use localStorage key: "themePreference".
Either way, apply immediately on load to avoid “flash” (FOUC).

IMPLEMENTATION STRATEGY
1) Token-based theming via CSS variables:
- Light tokens and Dark tokens must be defined in globals (e.g., globals.css)
- Use data attribute on <html> or <body>:
  - <html data-theme="light">
  - <html data-theme="dark">
- When "system", set data-theme based on prefers-color-scheme media query, and update on changes.

2) Add theme toggle icon button
- Place in top-right of the main app shell header (visible on all main pages)
- Use lucide icons:
  - Sun for light
  - Moon for dark
- Toggle behavior:
  - If current resolved theme is light -> switch to dark
  - If current resolved theme is dark -> switch to light
- When user toggles via icon, set preference explicitly (light/dark), not “system”.
- Add subtle tooltip: “Switch to Dark Mode” / “Switch to Light Mode”
- Add tiny, premium microinteraction (150–200ms fade/rotate, very subtle).

3) Settings selector
- In Settings page, add section: “Appearance”
- Add segmented control or radio buttons:
  - Light
  - Dark
  - System
- Show current selection.
- If “System” selected, show helper text:
  “Matches your device settings.”

4) Prevent flash on initial load
- Apply theme before render:
  - Inline small script in root layout OR
  - Use Next.js approach to set class/data attribute as early as possible.
- Ensure the page doesn’t flash light then dark.
