REPLIT AGENT — ADAPTIVE ENGINE v1 (Behavior → Next Plan) + ALWAYS USE PROFILE

Objective:
Implement Adaptive Engine v1: a deterministic backend layer that uses the user’s Profile + recent behavior (check-ins, adherence, energy, weight trend, plan changes) to automatically adjust future plan generation (Wellness, Meal, Workout, Nutrition, and Daily items if applicable). This must make the system feel “alive” and intelligent without adding new UI.

NON-NEGOTIABLE:
1) EVERY generation request MUST use Profile as baseline context (always).
   - Meal Plan, Workout Plan, Nutrition Plan, Wellness Plan, Daily Meal, Daily Workout.
   - If profile missing or incomplete → block generation with CTA to complete Profile.
2) Adaptive Engine must be deterministic (rule-based), not LLM-based.
3) Adaptive modifiers must be persisted and auditable (saved as JSON snapshot).
4) Adaptive modifiers must be injected into prompts/spec so the AI output changes measurably.
5) Plan snapshot must store:
   - profileSnapshot (merged profile + overrides)
   - adaptiveSnapshot (modifiers + reason codes)
   - safeSpec snapshot (if you already store it)

------------------------------------------------------------
PHASE 1 — Create Adaptive Engine module
------------------------------------------------------------
Create folder: server/adaptive/

Files:
1) server/adaptive/types.ts
Define:
- AdaptiveInputs: profile, latestPerformanceSummary, last2PerformanceSummaries, lastCheckIn, goal, pace, recentPlanChangeStats, currentSchedule
- AdaptiveModifiers:
  {
    volumeMultiplier: number,        // default 1.0 (0.85–1.15)
    intensityCapRPE: number,         // default 8, lower if fatigue
    cardioBias: "lower"|"normal"|"higher",
    recoveryBias: "normal"|"higher",
    complexityLevel: "simple"|"standard"|"advanced",
    nutritionCalorieDeltaKcal: number, // -300..+300 typical (bounded)
    trainingDayCarbBias: "lower"|"normal"|"higher",
    simplifyMeals: boolean,
    deloadWeek: boolean
  }
- AdaptiveDecision: { code: string, severity: "info"|"adjust", message: string }
- AdaptiveResult: { modifiers: AdaptiveModifiers, decisions: AdaptiveDecision[] }

2) server/adaptive/computeAdaptiveModifiers.ts
Deterministic rules (v1) using these signals:
- mealAdherencePct, workoutAdherencePct
- energyAvg (0–100)
- adherenceScore
- weightDeltaKg (if available)
- pace setting (gentle/steady/aggressive)
- recent regen/swap usage (plan-change volatility)

RULES (START SIMPLE & SAFE):

A) Base defaults:
- volumeMultiplier = 1.0
- intensityCapRPE = 8
- cardioBias = "normal"
- recoveryBias = "normal"
- complexityLevel = "standard"
- nutritionCalorieDeltaKcal = 0
- trainingDayCarbBias = "normal"
- simplifyMeals = false
- deloadWeek = false

B) Compliance rules (workouts):
- If workoutAdherencePct exists AND < 60:
    volumeMultiplier = 0.85
    complexityLevel = "simple"
    cardioBias = "lower"
    decisions += ["LOW_WORKOUT_ADHERENCE_SIMPLIFY"]
- If workoutAdherencePct 60–80:
    keep default
- If workoutAdherencePct > 85 AND energyAvg >= 80:
    volumeMultiplier = 1.05
    complexityLevel = min(currentLevel+1, "advanced") but only if experience is not beginner
    decisions += ["HIGH_WORKOUT_ADHERENCE_PROGRESS"]
- Cap volumeMultiplier within [0.85, 1.15]

C) Meal adherence rules:
- If mealAdherencePct exists AND < 60:
    simplifyMeals = true
    complexityLevel = "simple"
    decisions += ["LOW_MEAL_ADHERENCE_SIMPLIFY_MEALS"]

D) Energy / fatigue rules:
- If energyAvg <= 40 AND adherenceScore >= 70:
    recoveryBias = "higher"
    intensityCapRPE = 7
    deloadWeek = true
    volumeMultiplier = min(volumeMultiplier, 0.9)
    cardioBias = "lower"
    decisions += ["FATIGUE_RISK_DELOAD"]
- If energyAvg <= 40 AND adherenceScore < 70:
    recoveryBias = "higher"
    intensityCapRPE = 7
    complexityLevel = "simple"
    decisions += ["LOW_ENERGY_SIMPLIFY"]

E) Weight trend rules (apply only if weight data exists and goal relevant):
- If primaryGoal = "weight_loss":
    If weightDeltaKg >= -0.05 for 2 consecutive weeks AND adherenceScore >= 70:
        nutritionCalorieDeltaKcal = -150 (bounded by safety constraints)
        cardioBias = "higher"
        decisions += ["STALL_WEIGHT_LOSS_ADJUST"]
- If primaryGoal = "muscle_gain":
    If weightDeltaKg <= 0.05 for 2 consecutive weeks AND adherenceScore >= 70:
        nutritionCalorieDeltaKcal = +150
        decisions += ["STALL_MUSCLE_GAIN_ADJUST"]

F) Pace modifier (gentle/steady/aggressive):
- gentle: cap calorieDelta abs <= 100; cap volume change <= 1.05; prefer recoveryBias
- steady: cap calorieDelta abs <= 200; cap volume change <= 1.10
- aggressive: cap calorieDelta abs <= 300; cap volume change <= 1.15

G) Volatility / too many regens/swaps (if tracked):
- If regenUsedHigh OR swapUsedHigh:
    complexityLevel = "simple"
    decisions += ["HIGH_PLAN_VOLATILITY_SIMPLIFY"]

Return AdaptiveResult with decisions.

------------------------------------------------------------

   - profileSnapshot (canonical)

7) Post-validate output
8) Save plan with:
   - profileSnapshot
   - safeSpec snapshot (if available)
   - adaptiveSnapshot

- Profile must be included in prompt context every time even if overrides exist.
